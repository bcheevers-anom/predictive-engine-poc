UPDATE — 3 June 2026

Hi team — following up on the above and wanted to share a correction and a status update.

Apologies — we made an error in the original message. After re-reading the API docs more carefully and doing further testing, we did get a custom snapshot working. Specifically, using this payload format:

  POST /api/v1/snapshot/
  {
    "intel_version": 1,
    "custom_config": {
      "filter": "created_ts >= 2024-01-01T00:00:00",
      "format": "json_v2",
      "chunk_ioc_count": 250000,
      "use_org_whitelist": false
    }
  }

This is now running (snapshot ID 8551547) and has so far exported 4.5 million observable records
across 18 chunks — great progress, thank you.

The earlier failures were due to us sending the format parameter at the top level rather than inside
custom_config, and not including intel_version. Our mistake, not an issue with your setup.
